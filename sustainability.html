Interactive Urban Traffic Simulation

A data-driven simulation tool for exploring the impact of urban traffic transformation through interactive, real-world scenarios. This project investigates how infrastructure changes—such as the planned closure of a central highway bridge—can be transformed into an opportunity for sustainable mobility awareness, citizen engagement, and policy support through visual simulation and interactive storytelling.

Problem

Urban areas face increasing pressure from traffic congestion, noise pollution, and CO₂ emissions, intensified by limited green spaces in city centers. In Kempten (Allgäu), the B19 highway runs directly through the city core, contributing significantly to traffic load and environmental impact. The planned closure of the St.-Mang-Brücke introduces both challenges and opportunities: while commuters fear delays and longer routes, the city sees potential for a more sustainable redistribution of traffic. The core challenge is how to make these complex infrastructural changes understandable and meaningful for citizens and decision-makers.

Approach

We developed an interactive urban traffic simulation that transforms real traffic data into a visual, explorable system. Using traffic flow data from Kempten, the system simulates different scenarios such as road closures, rerouting strategies, and congestion patterns. Users can interact with the environment and immediately observe environmental and structural impacts. The simulation combines data-driven modeling with serious gamification and storytelling, making urban mobility planning more accessible.

System Design

The system is built around three core layers:

  1. Traffic Simulation Engine: based on real-world mobility data, including low- and peak-hour conditions, to simulate realistic traffic behavior.
  2. Interactive Visualization Layer: a map-based interface visualizing traffic flow and congestion, CO₂ emission heatmaps, noise intensity zones, and dynamic rerouting effects.
  3. User Interaction Layer: users can place road barriers, adjust traffic density, trigger scenario simulations, and observe real-time system responses.

Interaction Concept

The application is designed for exploratory urban storytelling. Users can test scenarios such as:

  • St.-Mang-Brücke closure scenario: traffic is rerouted through alternative paths, showing only minor increases in travel time while significantly reducing inner-city emissions and noise.
  • Residenzplatz closure scenario: a second experimental setup to analyze local traffic redistribution and environmental impact.
  • This transforms urban planning into an interactive cause-and-effect experience.

Key Features

  • Real-time traffic simulation based on OSM data.
  • CO₂, noise, and congestion visualizations.
  • Interactive road closure system (“Set Barrier”).
  • Scenario-based exploration of urban planning decisions.
  • Simplified interface designed for non-experts.
  • Serious gamification for sustainable decision-making.

Technology & Implementation

The system is built on a modular pipeline using open urban data and simulation tools. The workflow includes:

  • OpenStreetMap (OSM) data extraction via Overpass Turbo.
  • Preprocessing and network refinement using JOSM.
  • Traffic simulation using SUMO.
  • GeoJSON conversion for visualization.
  • React + Vite frontend for interactive map rendering.
  • Heatmaps for emissions, noise, and congestion.
  • Scenario-based simulation control designed to be scalable and reusable for other cities.

Conclusion

    Our tool demonstrates how urban infrastructure planning can be transformed into an accessible, interactive, and data-driven experience. By combining traffic simulation, visualization, and storytelling, the project helps make abstract infrastructural decisions tangible for the public. As cities move toward more sustainable futures, such tools can act as a bridge between data, policy, and citizen engagement.
